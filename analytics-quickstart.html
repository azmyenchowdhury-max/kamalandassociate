🎯 How to Use Analytics

View Real-Time Dashboard

Visit the analytics dashboard to see:

  • Total sessions (last 100 tracked)
  • Total pageviews and conversions
  • Device distribution (mobile/tablet/desktop %)
  • Traffic sources (Google, Facebook, direct, etc.)
  • Real-time Core Web Vitals scores
  • Page load breakdown (DNS, TCP, TTFB, total)
  • Performance score (0-100)
  • Recent sessions list

Refreshes automatically every 10 seconds

Check Console Analytics (DevTools)

Press F12 to open DevTools, go to Console tab, and run these commands:

View Current Session Summary:

window.__analyticsDashboard.getAnalyticsSummary();

View Last 10 Sessions:

window.__analyticsDashboard.getStoredSessions(10);

View Aggregate Metrics (All Sessions):

window.__analyticsDashboard.getAggregateMetrics();

View Web Vitals Data:

window.__webVitalsData;

Log Full Analytics Summary:

window.__analyticsDashboard.logAnalyticsSummary();

Get Current Session ID:

window.__analyticsDashboard.sessionId;

Get Device Type:

window.__analyticsDashboard.deviceType;

⚙️ Setup Configuration

Configure Google Analytics 4

To connect to Google Analytics 4 dashboard:

  1. Get your GA4 Measurement ID from Google Analytics
  2. Open js/runtime-config.js
  3. Find line with window.__GA_MEASUREMENT_ID__
  4. Replace placeholder with your ID: "G-XXXXXXXXXX"
  5. Save file and reload browser
  6. Events will appear in GA4 dashboard within minutes

🔧 Troubleshooting

Analytics Not Showing Data?

Check 1: Open DevTools (F12) → Console

Check 2: Run: window.__analyticsDashboard.sessionId

Check 3: Verify localStorage enabled: localStorage.setItem('test', '1')

Check 4: Ensure not in Private/Incognito mode (localStorage disabled)

Check 5: Clear old data: localStorage.removeItem('kamal_analytics_sessions')

Low Performance Scores?

Check which metric is low (LCP, FID, CLS)

Use Chrome DevTools Performance tab to profile

Optimize largest images first (biggest impact on LCP)

Minify CSS/JavaScript to reduce FID

Reserve space for images to improve CLS

📊 Key Metrics Explained

📌 LCP (Largest Contentful Paint)

Time when the largest visible element finishes rendering.

Target: < 2.5 seconds

Impact: User perceives how fast page loads

Improve: Optimize images, minify code, use caching

✌️ FID (First Input Delay)

Time between user interaction and browser response.

Target: < 100 milliseconds

Impact: Page feels responsive to user clicks

Improve: Break up long tasks, defer JavaScript

📐 CLS (Cumulative Layout Shift)

Measure of unexpected layout changes during page load.

Target: < 0.1

Impact: Page layout stable (no jumping content)

Improve: Add width/height to images, reserve ad space

🎬 Page Load Time

Total time from navigation start until fully loaded.

Target: < 3 seconds

Impact: Overall page performance perception

Improve: CDN, compression, lazy loading, minification
